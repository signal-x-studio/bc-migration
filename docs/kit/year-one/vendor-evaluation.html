Semantic Layer Vendor Evaluation

Scoring rubric for vector databases, event streaming, and semantic layer platforms

Date: January 2026 Related: ADR-001 Timeline: 8 weeks to selection

Evaluation Framework Overview

What We're Evaluating

Vector Database

Store and query embeddings for semantic search and recommendations

Pinecone Weaviate Qdrant Milvus pgvector

Event Streaming

Real-time data backbone with CDC for keeping systems in sync

Confluent Kinesis Redpanda MSK

Semantic Layer

Query abstraction and metrics layer for unified data access

Cube Materialize dbt SL AtScale

Scoring System

Each criterion is scored 1-5:

1
Does not meet
2
Partial with gaps
3
Meets basic
4
Exceeds
5
Best-in-class

POC Requirements

Dataset: 1M product vectors with metadata (category, price, merchant ID)

Similarity search (top-10 similar products)
Filtered search (category + price range)
Bulk insert (100K vectors in batch)
Real-time insert latency test
Delete and re-query consistency
Success Criteria:
  • P99 query latency <100ms at 1M vectors
  • Filtered search doesn't degrade >2x vs. unfiltered
  • Bulk insert completes in <10 minutes
  • Real-time insert latency <50ms

Event Streaming Evaluation

ES
Event Streaming Candidates
Vendor Type Notes
Confluent Cloud Managed Kafka Enterprise leader, rich ecosystem
AWS Kinesis AWS Native Good if AWS-heavy
Redpanda Kafka-compatible Performance focus, simpler ops
AWS MSK Managed Kafka on AWS Middle ground
Technical Capabilities 40% weight
Criterion Weight What We're Looking For
Throughput 10% Handle 100K+ events/second
Latency 8% P99 <50ms end-to-end
Durability 8% Zero data loss guarantees
Exactly-once Semantics 6% Critical for data consistency
Schema Registry 4% Native or integrated schema management
Connectors Ecosystem 4% Pre-built connectors for our data sources
CDC & Integration 20% weight
Criterion Weight What We're Looking For
CDC Capabilities 8% Debezium or native CDC support
Database Connectors 6% Postgres, MySQL, MongoDB support
Sink Connectors 6% Connect to vector DB, data warehouse
Critical: CDC is how we feed the semantic layer. Test actual CDC setup with our database types.

POC Requirements

Setup: CDC from test Postgres, 3-topic architecture, consumer writing to vector DB

Steady-state: 10K events/second
Burst: 100K events in 1 minute
Consumer lag recovery
Schema evolution test
Failover simulation
Success Criteria:
  • Sustained 10K events/second with <50ms latency
  • Burst absorbed without data loss
  • Consumer catches up within 5 minutes after 1-hour pause
  • Schema evolution works without consumer restart

Criterion Weight What We're Looking For
Vector DB Integration 10% Native or easy integration with our vector DB

Make-or-break: If can't integrate with vector DB, it's a no-go. Test actual integration, not just claimed support.

POC Requirements

Setup: Connect to vector DB and event stream, define 3-5 semantic models

API query for product recommendations
Real-time metric update (new order)
Cross-source query (vector + structured)
Caching effectiveness test
Success Criteria:
  • API queries return in <200ms
  • Real-time updates reflected within 1 second
  • Cross-source queries work correctly
  • Cached queries return in <50ms

Must-Haves (Elimination Criteria)

If a vendor fails any of these, they're eliminated:

All Categories SOC2 compliance or clear path to it
All Categories Managed service option (no self-hosting requirement)
Vector DB Hybrid search support + 10M+ vector scale demonstrated
Event Streaming CDC support (native or Debezium) + exactly-once semantics
Semantic Layer Real-time source support + API access (not just BI tool)
