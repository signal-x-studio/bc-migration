API Clients Architecture

The system provides dedicated API clients for WooCommerce and BigCommerce with rate limiting, retry logic, and error handling.

WCClient

WooCommerce REST API client (src/assessment/wc-client.ts):

  • OAuth 1.0a authentication
  • Automatic pagination
  • Error handling and retry logic
  • Type-safe responses

BCClient

BigCommerce REST API client (src/bigcommerce/bc-client.ts):

  • OAuth 2.0 token authentication
  • Rate limiting integration
  • Automatic retry on transient failures
  • Type-safe responses

Common Patterns

Request Wrapping

// All API calls are wrapped with rate limiting and retry
const product = await withRetry(
  () => withRateLimit(() => bcClient.getProduct(productId))
);

Error Handling

try {
  const product = await bcClient.getProduct(productId);
} catch (error) {
  const apiError = ApiError.fromAxiosError(error, { productId });
  if (apiError.retriable) {
    // Will be retried automatically
  }
}

Best Practices

  • Always use rate limiting
  • Implement retry logic for transient failures
  • Handle errors with context
  • Use type-safe responses
  • Monitor API usage and limits
