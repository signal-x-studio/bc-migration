The "Keep WordPress" SEO Advantage

The core architectural decision of the BC Migration approach—keeping WordPress as the frontend while migrating commerce to BigCommerce—provides significant SEO advantages over full replatforming.

What Stays in WordPress (SEO Intact)

  • All content URLs remain unchanged (blog posts, pages, media)
  • SEO plugins (Yoast, RankMath) continue working
  • Existing internal linking structure preserved
  • Meta descriptions, titles, and schema markup unchanged
  • XML sitemaps for content continue as-is
  • Canonical URLs for content pages unchanged

Comparison: Keep WordPress vs. Full Replatform

SEO Preservation in Migration Stages

WPE's Artemis framework includes SEO Data Preservation as an explicit phase in Stage 1 of migration.

1 Pre-Migration Audit

Before migration, capture baseline SEO data:

  • Export all product URLs from WooCommerce
  • Document current product meta titles and descriptions
  • Capture Google Search Console rankings for product pages
  • Note any custom URL structures (/shop/, /product/, etc.)

2 URL Mapping Strategy

Plan URL handling for product pages:

  • BC Bridge plugin handles product routing from WordPress URLs
  • Decide: Keep WordPress URL structure OR adopt BigCommerce patterns
  • Generate redirect rules for any URL changes

3 Product Data Migration with SEO Fields

The bc-migration tool preserves SEO data during product migration:

  • Product titles → BigCommerce product name
  • Meta descriptions → BigCommerce meta description
  • URL slugs → BigCommerce custom URL (when possible)
  • Image alt text preserved during image migration

4 Post-Migration Verification

After migration, validate SEO integrity:

  • Verify all product pages render with correct meta tags
  • Test canonical URLs resolve correctly
  • Confirm structured data (schema.org) is present
  • Submit updated XML sitemap to Search Console
  • Monitor rankings for key product pages

Product URL Strategy

The BC Bridge plugin provides URL routing for product pages. There are two primary approaches:

Option A: WordPress-Native URLs

Products accessed via WordPress URL structure, with BC Bridge fetching data from BigCommerce API.

WordPress URL:  /product/blue-widget/
BC Bridge:      Fetches product data from BigCommerce API
Checkout:       Redirects to BigCommerce checkout

SEO Advantage:  URLs unchanged from WooCommerce

Option B: BigCommerce-Proxied URLs

Products use BigCommerce URL structure, proxied through WordPress for consistent frontend.

WordPress URL:  /shop/blue-widget/  (custom rewrite)
BC Bridge:      Routes to BigCommerce product
Checkout:       BigCommerce native

SEO Note:       Requires redirects from old WooCommerce URLs

Recommendation

For maximum SEO preservation, use Option A with existing WooCommerce URL structure. The BC Bridge plugin maintains URL compatibility while fetching commerce data from BigCommerce.

SEO Considerations Requiring Attention

While the "Keep WordPress" approach preserves most SEO, some areas need explicit handling:

SEO Element Risk Level Mitigation
Product page URLs Medium BC Bridge maintains URL compatibility; set up 301 redirects for any changes
Product schema markup Low BigCommerce generates Product schema; verify on migration
Category page URLs Medium Plan category URL structure; implement redirects as needed
Internal links to products Low Audit and update internal links in WordPress content
Product XML sitemap Low Generate new product sitemap from BigCommerce; submit to Search Console

From Granite Analysis

"Product SEO preservation is challenging when migrating from WooCommerce" — this is why the Keep WordPress approach is recommended: it preserves content SEO entirely and only requires attention to product-specific SEO elements.
