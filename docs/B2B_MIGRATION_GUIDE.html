B2B Migration Guide

Migrating B2B WooCommerce stores to BigCommerce B2B Edition

Granite Gap Analysis

Source: Granite Comparative Analysis Slides 66-73, 174-182 | Last Updated: December 2025

Executive Summary

B2B migrations are significantly more complex than B2C due to custom pricing, customer groups, quoting workflows, and ERP integrations. This guide addresses the specific considerations for migrating WooCommerce B2B stores to BigCommerce B2B Edition.

Complexity Warning

B2B migrations typically require 2-3x the effort of equivalent B2C migrations due to pricing complexity, approval workflows, and integration requirements. Plan accordingly.

BigCommerce B2B Edition

Native B2B capabilities including price lists, customer groups, quoting, and purchase orders. Enterprise-grade solution.

WooCommerce B2B Landscape

Fragmented plugin ecosystem: B2BKing, WooCommerce B2B, Wholesale Suite. Each with different data models.

Migration Complexity

Custom pricing rules, approval workflows, and ERP integrations require careful mapping and potential rebuilding.

Price List Migration Strategy

B2B pricing is the most complex aspect of migration. BigCommerce uses a Price Lists system that differs from WooCommerce's plugin-based approach.

WooCommerce B2B Pricing Models

  • Role-Based Pricing: WordPress user roles (wholesale_customer, vip_customer) with percentage discounts
  • Customer-Specific: Per-customer prices stored in user meta or custom tables
  • Tiered Pricing: Quantity-based price breaks (buy 10+ get 10% off)
  • Category Discounts: Wholesale pricing by product category
  • Negotiated Contracts: Custom pricing per customer, often in spreadsheets or ERP

BigCommerce Price Lists Architecture

  • Price Lists: Named pricing configurations assignable to customer groups
  • Price Records: SKU-level price overrides within a price list
  • Bulk Pricing: Native quantity-based pricing rules
  • Customer Group Assignment: Map customers to price lists via groups

Migration Approach

Extract WooCommerce pricing data → Transform to BC Price List format → Import via Price Lists API → Validate pricing accuracy → Assign customer groups

Data Transformation Example

WooCommerce Source BigCommerce Target Transformation

Quote Workflow Migration

B2B quoting (RFQ) is a common WooCommerce customization that maps well to BigCommerce's native quoting system.

WooCommerce Quoting Landscape

  • YITH Request a Quote: Most popular, stores quotes as custom post type
  • B2BKing Quotes: Integrated with B2BKing B2B suite
  • Custom Implementations: Many stores have bespoke quoting systems

BigCommerce Native Quoting

  • Quote creation from cart or by sales rep
  • Quote approval workflows
  • Quote-to-order conversion
  • Quote expiration and versioning
  • API access for integration

Historical Quote Data

Historical WooCommerce quotes are typically NOT migrated. Consider exporting to PDF/archive for reference. New quotes start fresh in BigCommerce.

Company Account Structure

B2B stores often have company accounts with multiple buyers, approval hierarchies, and shared carts.

Migration Considerations

WooCommerce Pattern BigCommerce B2B Migration Notes
Individual WordPress users Company → Users Group users into companies
Company stored in user meta Company entity Extract and create companies
User roles for permissions Buyer roles (Admin, Buyer, etc.) Map roles appropriately
B2BKing company accounts Companies API Direct structure migration

Buyer Role Mapping

Typical WooCommerce Role BigCommerce B2B Role Capabilities
Company Admin Company Admin Manage users, view all orders, full access
Purchasing Manager Senior Buyer Approve orders, manage budgets
Buyer Junior Buyer Create orders, require approval
View Only Custom Role Browse catalog, no purchasing

ERP Integration Considerations

B2B stores commonly integrate with ERP systems. These integrations must be rebuilt for BigCommerce.

Critical: ERP Integration Rebuild

ERP integrations (NetSuite, SAP, Microsoft Dynamics, etc.) require complete rebuilding. Plan for this as a separate workstream with dedicated timeline.

Common ERP Integration Points

Data Flow WooCommerce Approach BigCommerce Approach
Product/Pricing Sync Plugins, WP Cron, custom Catalog API, webhooks
Inventory Sync Real-time or scheduled Inventory API, webhooks
Customer Sync User meta, custom tables Customers API, Companies API
Order Push Order hooks, plugins Orders API, webhooks
Invoice/Payment Payment gateway callbacks Payment webhooks, API

BigCommerce ERP Connectors

BigCommerce has native connectors and partnerships for major ERPs:

  • NetSuite: Native SuiteCommerce integration
  • Microsoft Dynamics: Celigo, other iPaaS
  • SAP: Partner integrations available
  • Acumatica: Native connector
  • QuickBooks: Native integration

B2B Migration Checklist

  • ☐ Inventory all B2B plugins in use
  • ☐ Document customer group structure
  • ☐ Export all pricing rules and price lists
  • ☐ Map company account hierarchy
  • ☐ Document quote/RFQ workflows
  • ☐ Identify all ERP integration points
  • ☐ Calculate B2B complexity score

Data Migration

  • ☐ Create customer groups in BigCommerce
  • ☐ Build price lists for each pricing tier
  • ☐ Migrate company accounts
  • ☐ Assign users to companies and roles
  • ☐ Configure payment terms
  • ☐ Set up catalog visibility rules

Integration Rebuild

  • ☐ Rebuild ERP product sync
  • ☐ Rebuild ERP inventory sync
  • ☐ Rebuild ERP order push
  • ☐ Configure BigCommerce webhooks
  • ☐ Test bi-directional data flow

Validation

  • ☐ Verify pricing accuracy for each customer group
  • ☐ Test tiered pricing calculations
  • ☐ Validate quote workflow end-to-end
  • ☐ Confirm company account permissions
  • ☐ Test payment terms functionality
  • ☐ Verify ERP sync accuracy
