ADR-001: Keep WordPress Architecture

Accepted

Context

When migrating stores from WooCommerce to BigCommerce, the traditional approach is a complete replatforming where both content management and commerce move to the new platform. However, this approach has significant drawbacks:

  1. Content Migration Complexity: WordPress sites often have years of blog content, landing pages, and custom designs built with page builders (Elementor, WPBakery, etc.)
  2. SEO Risk: WooCommerce's WordPress foundation delivers 92% SEO scores—the highest in the industry. Full replatforming risks losing hard-won organic search equity
  3. Team Disruption: Marketing teams know WordPress; forcing them to learn BigCommerce's CMS creates friction
  4. Plugin Ecosystem Loss: WordPress's 50,000+ plugin ecosystem would need replacement
  5. Design Investment: Custom themes and page builder designs represent significant investment

Decision

We will preserve WordPress as the content and presentation layer while migrating commerce functionality to BigCommerce.

This "headless commerce" architecture uses:

  • WordPress: Handles content, themes, SEO, page building, and user experience
  • BigCommerce: Handles products, checkout, payments, inventory, and order processing
  • BC Bridge Plugin: Connects the two platforms via BigCommerce APIs

Consequences

Positive

  • SEO Preservation: WordPress URLs, content, and SEO plugins remain unchanged
  • Zero Content Migration: Blog posts, pages, and designs stay in place
  • Team Continuity: Marketing can continue using familiar WordPress tools
  • Plugin Preservation: ~30% of WooCommerce plugins remain functional
  • Reduced Migration Scope: Only commerce data moves
  • Faster Time-to-Value: Migration timeline reduced

Negative

  • Dual System Maintenance: Must maintain both WordPress and BigCommerce
  • Integration Complexity: BC Bridge plugin adds a synchronization layer
  • Checkout Handoff: Users transition to BigCommerce hosted checkout
  • Product SEO Challenge: Product page SEO requires redirect handling

Neutral

  • Learning Curve: Teams need to learn BigCommerce admin for commerce operations
  • Cost Structure: Trading WooCommerce hosting/plugin costs for BigCommerce subscription

Alternatives Considered

Alternative 1: Full BigCommerce Replatform

Move everything to BigCommerce, abandon WordPress.

Rejected: Loses SEO advantage, requires content migration, disrupts teams

Alternative 2: Full Shopify Replatform

Move to Shopify as the dominant player.

Rejected: Higher costs (transaction fees), weaker B2B, loses WordPress content investment

Alternative 3: Stay on WooCommerce

Optimize current WooCommerce installation.

Rejected: Doesn't address core scalability, security, and maintenance burden issues
