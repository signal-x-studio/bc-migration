BC Bridge Templates Guide

Complete guide to the BC Bridge template system, including available templates, template hierarchy, and customization options.

Template Hierarchy

BC Bridge follows WordPress template hierarchy. Templates are loaded in this order:

  1. Theme Override: your-theme/bc-bridge/template-name.php
  2. Plugin Default: bc-bridge/templates/template-name.php

Available Templates

shop.php

Shop page template - displays product listing.

Location: bc-bridge/templates/shop.php

Variables Available:

  • $products - Array of product data
  • $pagination - Pagination data

    • product.php

    Single product page template.

    Location: bc-bridge/templates/product.php

    Variables Available:

    • $product - Product data array
    • $variants - Product variants (if applicable)

      • category.php

      Category archive page template.

      Location: bc-bridge/templates/category.php

      Variables Available:

      • $category - Category data
      • $products - Products in category

        • cart.php

        Shopping cart page template.

        Location: bc-bridge/templates/cart.php

        Variables Available:

        • $cart - Cart data array
        • $items - Cart items array

          • checkout.php

          Checkout page template.

          Location: bc-bridge/templates/checkout.php

          Variables Available:

          • $cart - Cart data
          • $checkout_url - BigCommerce checkout URL

            • Template Partials

            partials/product-card.php

            Product card component - used in product grids.

            Usage:

            get_template_part('bc-bridge/partials/product-card', null, ['product' => $product]);

            partials/product-grid.php

            Product grid component - displays multiple products.

            Usage:

            get_template_part('bc-bridge/partials/product-grid', null, ['products' => $products]);

            partials/error.php

            Error message component.

            Usage:

            get_template_part('bc-bridge/partials/error', null, ['message' => $error_message]);

            Template Functions

            bc_bridge_get_product()

            Get current product data.

            $product = bc_bridge_get_product();

            bc_bridge_get_products()

            Get products with optional filters.

            $products = bc_bridge_get_products([
    'limit' => 12,
    'page' => 1,
    'categories:in' => 123,
]);

            bc_bridge_get_category()

            Get current category data.

            $category = bc_bridge_get_category();

            bc_bridge_get_cart()

            Get current cart data.

            $cart = bc_bridge_get_cart();

            Overriding Templates

            Step 1: Create Directory

            mkdir -p your-theme/bc-bridge

            Step 2: Copy Template

            cp bc-bridge/templates/shop.php your-theme/bc-bridge/shop.php

            Step 3: Customize

            // your-theme/bc-bridge/shop.php
<?php
get_header();

$products = bc_bridge_get_products(['limit' => 12]);
?>

<div class="custom-shop">
    <h1>Our Products</h1>
    
    <?php if (!empty($products)) : ?>
        <div class="product-grid">
            <?php foreach ($products as $product) : ?>
                <?php get_template_part('bc-bridge/partials/product-card', null, ['product' => $product]); ?>
            <?php endforeach; ?>
        </div>
    <?php endif; ?>
</div>

<?php get_footer(); ?>

            Template Hooks

            Templates include action hooks for customization:

            // Before product content
do_action('bc_bridge_before_product_content', $product);

// After product content
do_action('bc_bridge_after_product_content', $product);

// Before product grid
do_action('bc_bridge_before_product_grid', $products);

// After product grid
do_action('bc_bridge_after_product_grid', $products);

            Best Practices

            • Always copy templates to your theme before modifying
            • Use template partials for reusable components
            • Follow WordPress template standards
            • Test templates after plugin updates
            • Use hooks instead of modifying core templates
