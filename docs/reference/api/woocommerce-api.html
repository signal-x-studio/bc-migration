WooCommerce REST API Reference

API endpoints used by the BC Migration tool to read data from WooCommerce stores.

Authentication

WooCommerce REST API uses HTTP Basic Authentication with Consumer Key and Consumer Secret:

Authorization: Basic base64(consumer_key:consumer_secret)

Or as query parameters:

?consumer_key=ck_xxx&consumer_secret=cs_xxx

Base URL

https://your-store.com/wp-json/wc/v3

Endpoints Used

Products

Get Products

GET /products

Query Parameters:

  • per_page - Number of products per page (default: 10, max: 100)
  • page - Page number for pagination
  • type - Filter by product type (simple, variable, grouped, external)
  • status - Filter by status (draft, pending, private, publish)

Response Headers:

  • X-WP-Total - Total number of products
  • X-WP-TotalPages - Total number of pages

Example:

curl "https://store.com/wp-json/wc/v3/products?per_page=1" \
  -u "ck_xxx:cs_xxx"

Get Product Count

GET /products?per_page=1

Check X-WP-Total header for total count without fetching all products.

Categories

Get Categories

GET /products/categories

Query Parameters:

  • per_page - Number of categories per page
  • page - Page number
  • parent - Filter by parent category ID
  • hide_empty - Hide empty categories (true/false)

Response Headers:

  • X-WP-Total - Total number of categories

Example:

curl "https://store.com/wp-json/wc/v3/products/categories" \
  -u "ck_xxx:cs_xxx"

Customers

Get Customers

GET /customers

Query Parameters:

  • per_page - Number of customers per page
  • page - Page number
  • role - Filter by customer role
  • email - Filter by email address

Response Headers:

  • X-WP-Total - Total number of customers

Example:

curl "https://store.com/wp-json/wc/v3/customers?per_page=1" \
  -u "ck_xxx:cs_xxx"

Orders

Get Orders

GET /orders

Query Parameters:

  • per_page - Number of orders per page
  • page - Page number
  • status - Filter by order status
  • customer - Filter by customer ID
  • after - Filter orders after date (ISO 8601)
  • before - Filter orders before date (ISO 8601)

Response Headers:

  • X-WP-Total - Total number of orders

Example:

curl "https://store.com/wp-json/wc/v3/orders?per_page=1" \
  -u "ck_xxx:cs_xxx"

System Status

Get System Status

GET /system_status

Returns WooCommerce and WordPress system information including:

  • WooCommerce version
  • WordPress version
  • PHP version
  • Database information
  • Active plugins

Example:

curl "https://store.com/wp-json/wc/v3/system_status" \
  -u "ck_xxx:cs_xxx"

Rate Limits

WooCommerce REST API has no hard rate limits, but:

  • Use pagination for large datasets
  • Implement reasonable request spacing
  • Check X-WP-Total header to avoid fetching unnecessary data

Error Responses

401 Unauthorized

{
  "code": "woocommerce_rest_authentication_error",
  "message": "Invalid API credentials"
}

Solutions:

  • Verify Consumer Key and Secret
  • Check API key permissions
  • Ensure REST API is enabled

404 Not Found

{
  "code": "rest_no_route",
  "message": "No route was found matching the URL and request method"
}

Solutions:

  • Verify WooCommerce is installed and active
  • Check REST API is enabled
  • Verify endpoint URL
