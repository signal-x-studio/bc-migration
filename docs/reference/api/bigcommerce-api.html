BigCommerce REST API Reference

API endpoints used by the BC Migration tool to write data to BigCommerce stores.

Authentication

BigCommerce REST API uses OAuth 2.0 with Access Token:

X-Auth-Token: your-access-token

Base URL

https://api.bigcommerce.com/stores/{store_hash}/v3

Where {store_hash} is your BigCommerce store hash.

Rate Limits

Standard Plan:

  • 150 requests per 30 seconds
  • 4 requests per second

Enterprise Plan:

  • Higher limits (varies by plan)

The migration tool handles rate limiting automatically with:

  • Request queuing
  • Exponential backoff
  • Rate limit header detection

Endpoints Used

Products

Create Products

POST /catalog/products

Batch Limit: 10 products per request

Request Body:

[
  {
    "name": "Product Name",
    "type": "physical",
    "weight": 1.5,
    "price": "29.99",
    "categories": [5],
    "images": [
      {
        "image_url": "https://cdn.bigcommerce.com/..."
      }
    ],
    "variants": [...]
  }
]

Response:

{
  "data": [
    {
      "id": 123,
      "name": "Product Name",
      ...
    }
  ],
  "meta": {}
}

Get Products

GET /catalog/products

Query Parameters:

  • page - Page number
  • limit - Items per page (max: 250)
  • include - Include related resources (variants, images, etc.)

Update Product

PUT /catalog/products/{product_id}

Delete Product

DELETE /catalog/products/{product_id}

Categories

Create Categories

POST /catalog/categories

Request Body:

{
  "name": "Category Name",
  "parent_id": 0,
  "description": "Category description",
  "is_visible": true,
  "sort_order": 0
}

Response:

{
  "data": {
    "id": 5,
    "name": "Category Name",
    ...
  },
  "meta": {}
}

Get Categories

GET /catalog/categories

Query Parameters:

  • page - Page number
  • limit - Items per page
  • parent_id - Filter by parent category

Customers

Create Customers

POST /customers

Batch Limit: 10 customers per request

Request Body:

[
  {
    "email": "customer@example.com",
    "first_name": "John",
    "last_name": "Doe",
    "phone": "+1234567890",
    "company": "Company Name",
    "addresses": [...]
  }
]

Response:

{
  "data": [
    {
      "id": 789,
      "email": "customer@example.com",
      ...
    }
  ],
  "meta": {}
}

Get Customers

GET /customers

Query Parameters:

  • page - Page number
  • limit - Items per page
  • email - Filter by email

Orders

Create Orders

POST /orders

Request Body:

{
  "customer_id": 789,
  "status_id": 1,
  "billing_address": {...},
  "products": [
    {
      "product_id": 123,
      "quantity": 2,
      "price_inc_tax": "59.98"
    }
  ]
}

Store Information

Get Store

GET /store

Returns store information including:

  • Store name
  • Domain
  • Currency
  • Timezone

Example:

curl "https://api.bigcommerce.com/stores/{store_hash}/v3/store" \
  -H "X-Auth-Token: your-token"

Error Responses

401 Unauthorized

{
  "status": 401,
  "title": "Unauthorized",
  "type": "https://developer.bigcommerce.com/api-docs/getting-started/api-status-codes",
  "errors": {}
}

Solutions:

  • Verify Access Token is correct
  • Check token hasn't expired
  • Ensure token has required scopes

403 Forbidden

{
  "status": 403,
  "title": "Forbidden",
  "type": "https://developer.bigcommerce.com/api-docs/getting-started/api-status-codes",
  "errors": {}
}

Solutions:

  • Check API token has required scopes
  • Verify store hash is correct

429 Too Many Requests

{
  "status": 429,
  "title": "Too Many Requests",
  "type": "https://developer.bigcommerce.com/api-docs/getting-started/api-status-codes",
  "errors": {}
}

Solutions:

  • Tool handles this automatically
  • Wait for rate limit window to reset
  • Check response headers for retry-after

Required Scopes

API token must have these scopes:

  • store_v2_products - Read/write products
  • store_v2_products_read_only - Read products (alternative)
  • store_v2_customers - Read/write customers
  • store_v2_orders - Read/write orders
  • store_cart - Cart management
  • store_checkout - Checkout URLs
  • store_channel_settings - Channel information

Platform Limits

Product Limits

  • Variants per product: 600 (hard limit)
  • Images per product: No hard limit
  • Custom fields per product: 250

Batch Limits

  • Products: 10 per batch request
  • Customers: 10 per batch request
  • Categories: No batch endpoint (create individually)

Response Headers

Important headers to check:

  • X-Rate-Limit-Requests-Left - Requests remaining in window
  • X-Rate-Limit-Time-Reset-Ms - Time until window resets
  • X-Rate-Limit-Requests-Quota - Total requests per window
