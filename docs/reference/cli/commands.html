CLI Commands Reference

Complete reference for the BC Migration command-line interface.

Installation

npm install

Global Options

All commands support these options:

  • --help - Show command help
  • --version - Show version number

Commands

assess

Analyze a WooCommerce store for migration suitability.

npm run assess
# or
npx tsx src/cli.ts assess [options]

Options:

  • -u, --url <url> - WooCommerce store URL
  • -k, --key <key> - WooCommerce Consumer Key
  • -s, --secret <secret> - WooCommerce Consumer Secret

Environment Variables:

  • WC_URL - Store URL
  • WC_CONSUMER_KEY - Consumer Key
  • WC_CONSUMER_SECRET - Consumer Secret

Output:

  • Generates Markdown and JSON reports in ./reports/
  • Report includes readiness score, metrics, and recommendations

validate-target

Verify BigCommerce API credentials and store accessibility.

npx tsx src/cli.ts validate-target [options]

Options:

  • --hash <hash> - BigCommerce Store Hash
  • --token <token> - BigCommerce Access Token

Environment Variables:

  • BC_STORE_HASH - Store Hash
  • BC_ACCESS_TOKEN - Access Token

migrate

Perform the migration. Pre-requisites: Run 'assess' first.

npx tsx src/cli.ts migrate [options]

Options:

  • --type <type> - Migration type: categories, products, customers, orders, all (default: all)
  • --delta - Enable delta sync (incremental migration)
  • --url <url> - WooCommerce store URL
  • --key <key> - WooCommerce Consumer Key
  • --secret <secret> - WooCommerce Consumer Secret
  • --hash <hash> - BigCommerce Store Hash
  • --token <token> - BigCommerce Access Token

Examples:

# Migrate all data
npx tsx src/cli.ts migrate

# Migrate only products
npx tsx src/cli.ts migrate --type products

# Delta migration (only modified items)
npx tsx src/cli.ts migrate --delta

validate

Verify migration results and data integrity. Pre-requisites: Run 'migrate' first.

npx tsx src/cli.ts validate [options]

Options:

  • --url <url> - WooCommerce store URL
  • --key <key> - WooCommerce Consumer Key
  • --secret <secret> - WooCommerce Consumer Secret
  • --hash <hash> - BigCommerce Store Hash
  • --token <token> - BigCommerce Access Token
  • --sample <n> - Sample size for spot checks (default: 10)

Output:

  • Generates validation reports in ./reports/ directory
  • Includes JSON and Markdown formats

Example:

npx tsx src/cli.ts assess \
  --url https://mystore.com \
  --key ck_1234567890 \
  --secret cs_0987654321

validate-target

Verify BigCommerce API credentials and store accessibility.

npm run validate-target
# or
npx tsx src/cli.ts validate-target [options]

Options:

  • --hash <hash> - BigCommerce Store Hash
  • --token <token> - BigCommerce Access Token

Environment Variables:

  • BC_STORE_HASH - Store Hash
  • BC_ACCESS_TOKEN - Access Token

Output:

  • Displays store information if connection successful
  • Shows error if credentials invalid

Example:

npx tsx src/cli.ts validate-target \
  --hash abc12345 \
  --token vt_xyz789

migrate

Migrate data from WooCommerce to BigCommerce.

npm run migrate
# or
npx tsx src/cli.ts migrate [options]

Options:

  • --type <type> - Migrate specific data type (categories, products, customers, orders)
  • --delta - Only migrate items modified since last run

Environment Variables:

  • WC_URL, WC_CONSUMER_KEY, WC_CONSUMER_SECRET - WooCommerce credentials
  • BC_STORE_HASH, BC_ACCESS_TOKEN - BigCommerce credentials

Migration Order:

  1. Categories (required first)
  2. Products (depends on categories)
  3. Customers (independent)
  4. Orders (depends on products and customers, optional)

Examples:

# Migrate all data types
npm run migrate

# Migrate specific type
npx tsx src/cli.ts migrate --type=products
npx tsx src/cli.ts migrate --type=categories
npx tsx src/cli.ts migrate --type=customers
npx tsx src/cli.ts migrate --type=orders

# Delta migration (only changed items)
npx tsx src/cli.ts migrate --delta

Output:

  • Progress bar showing migration progress
  • Summary of items migrated
  • Warnings for skipped items
  • Logs saved to logs/migration-*.log

validate

Verify migration results and data integrity.

npm run validate
# or
npx tsx src/cli.ts validate [options]

Options:

  • --sample <number> - Number of samples to validate (default: 10)

Output:

  • Count comparisons (WC vs BC)
  • Sample data validation
  • Image accessibility checks
  • Validation report in console

Example:

# Default validation (10 samples)
npm run validate

# Custom sample size
npx tsx src/cli.ts validate --sample 20

Validates:

  • Product counts match
  • Category counts match
  • Customer counts match
  • Sample price accuracy
  • Image URLs accessible

Environment Variables

All commands can use environment variables from .env file:

# WooCommerce
WC_URL=https://your-store.com
WC_CONSUMER_KEY=ck_xxxxx
WC_CONSUMER_SECRET=cs_xxxxx

# BigCommerce
BC_STORE_HASH=your-store-hash
BC_ACCESS_TOKEN=your-api-token

Exit Codes

  • 0 - Success
  • 1 - Error (connection, validation, migration failure)

Logging

All commands log to:

  • Console (stdout/stderr)
  • Log files in logs/ directory
  • Format: migration-YYYY-MM-DDTHH-mm-ss.log

Log levels:

  • info - General information
  • warn - Warnings (non-fatal)
  • error - Errors (fatal)
