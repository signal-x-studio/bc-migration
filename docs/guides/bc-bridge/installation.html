BC Bridge Installation Guide

Step-by-step guide to installing the BC Bridge WordPress plugin.

Prerequisites

Before installing BC Bridge, ensure you have:

  1. WordPress Site

    • WordPress 6.2 or higher
    • PHP 8.1 or higher
    • WooCommerce deactivated (or ready to deactivate)

  2. BigCommerce Store

    • BigCommerce account created
    • Store configured and products migrated
    • API credentials (Store Hash, Client ID, Access Token)

  3. BigCommerce Channel

    • Channel created for WordPress site
    • Domain configured (if using custom domain)

Installation Methods

Method 1: WordPress Admin (Recommended)

  1. Download Plugin

    • Download BC Bridge plugin ZIP file
    • Or get from WordPress.org (when available)

  2. Install Plugin

    • Go to WordPress Admin → Plugins → Add New
    • Click "Upload Plugin"
    • Choose the BC Bridge ZIP file
    • Click "Install Now"

  3. Activate Plugin

    • After installation, click "Activate Plugin"
    • Plugin will activate and show setup wizard

Method 2: Manual Installation

  1. Upload Files

    • Extract BC Bridge ZIP file
    • Upload bc-bridge folder to /wp-content/plugins/

  2. Activate Plugin

    • Go to WordPress Admin → Plugins
    • Find "BC Bridge" in plugin list
    • Click "Activate"

Initial Setup

After activation, BC Bridge will guide you through setup:

Step 1: API Credentials

Enter your BigCommerce API credentials:

  • Store Hash: Your BigCommerce store hash
  • Client ID: Your BigCommerce API client ID
  • Access Token: Your BigCommerce API access token

Where to find these:

  1. Go to BigCommerce Admin → Settings → API Accounts
  2. Create new API account (or use existing)
  3. Copy Store Hash, Client ID, and Access Token

Step 2: Channel Selection

Select which BigCommerce channel to use:

  • Choose from your available channels
  • Or create a new channel for WordPress

Step 3: Test Connection

BC Bridge will test the connection:

  • Success: Connection verified, ready to use
  • Failure: Check credentials and try again

Step 4: Route Configuration

Configure which routes BC Bridge handles:

  • /shop - Product listing page
  • /product/* - Product detail pages
  • /cart - Shopping cart
  • /checkout - Checkout flow

Post-Installation

Deactivate WooCommerce

If WooCommerce is still active:

  1. Go to Plugins → Installed Plugins
  2. Find WooCommerce
  3. Click "Deactivate"
  4. BC Bridge will handle all commerce functionality

Note: Your WooCommerce data stays in the database. Only the plugin is deactivated.

Verify Installation

Test that BC Bridge is working:

  1. Visit Shop Page

    • Go to /shop on your site
    • Should see BigCommerce products

  2. View Product

    • Click on a product
    • Should see product details from BigCommerce

  3. Test Cart

    • Add product to cart
    • Cart should work with BigCommerce

Troubleshooting

Plugin Won't Activate

Check:

  • WordPress version (6.2+ required)
  • PHP version (8.1+ required)
  • Plugin file permissions
  • Error logs for specific errors

Connection Test Fails

Check:

  • API credentials are correct
  • Store Hash has no dashes
  • Access Token has required scopes
  • BigCommerce store is active

Products Not Showing

Check:

  • BigCommerce store has products
  • Channel is selected correctly
  • Route interception is working
  • Check browser console for errors
