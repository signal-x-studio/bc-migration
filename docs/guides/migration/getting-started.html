Migration Getting Started

Quick start guide for migrating your WooCommerce store to BigCommerce.

Prerequisites

Before starting migration, ensure you have:

  1. WooCommerce Store Access

    • REST API credentials (Consumer Key/Secret)
    • Admin access to verify data

  2. BigCommerce Store

    • BigCommerce account created
    • Store Hash and Access Token
    • Store configured (currency, tax, shipping)

  3. Migration Tool

    • Node.js 18+ installed
    • BC Migration tool installed (npm install)

Quick Start

1. Assess Your Store

First, run an assessment to understand your store:

npm run assess

Review the report in ./reports/ to identify:

  • Data volume (products, customers, orders)
  • Complexity indicators
  • Potential blockers

2. Configure Credentials

Set up your .env file:

# WooCommerce
WC_URL=https://your-store.com
WC_CONSUMER_KEY=ck_xxxxx
WC_CONSUMER_SECRET=cs_xxxxx

# BigCommerce
BC_STORE_HASH=your-store-hash
BC_ACCESS_TOKEN=your-api-token

3. Validate Connections

Test both connections:

# Test BigCommerce
npm run validate-target

# Test WooCommerce (runs assessment)
npm run assess

4. Run Migration

Migrate all data types:

npm run migrate

Or migrate specific types:

# Categories first (required for products)
npx tsx src/cli.ts migrate --type=categories

# Then products
npx tsx src/cli.ts migrate --type=products

# Then customers
npx tsx src/cli.ts migrate --type=customers

# Finally orders (optional)
npx tsx src/cli.ts migrate --type=orders

5. Validate Results

Verify migration completeness:

npm run validate

This checks:

  • Product counts match
  • Category counts match
  • Customer counts match
  • Sample data accuracy

Migration Order

Data must be migrated in this order:

  1. Categories - Required before products
  2. Products - Depends on categories
  3. Customers - Independent
  4. Orders - Depends on products and customers (optional)

What Gets Migrated

Data Type Migrated? Notes
Categories Yes With hierarchy and descriptions
Products Yes Including variants, images, prices
Product Variants Yes As part of products
Customers Yes Password reset required
Orders Optional Historical orders
Coupons Partial Basic coupons only
Reviews Partial Product reviews

What Doesn't Get Migrated

  • WordPress content (pages, posts, media)
  • WooCommerce settings (configured in BC)
  • Payment gateway settings (configured in BC)
  • Shipping methods (configured in BC)
  • Tax settings (configured in BC)
  • Custom post types (stays in WordPress)
