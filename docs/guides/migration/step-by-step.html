Step-by-Step Migration Guide

Complete guide to migrating your WooCommerce store to BigCommerce.

Phase 1: Preparation

1.1 Assess Your Store

Run a comprehensive assessment:

npm run assess

Review the report:

  • Check readiness score
  • Identify blockers (RED items)
  • Review warnings (YELLOW items)
  • Note complexity indicators

Key metrics to review:

  • Product count and variants
  • Customer count
  • Order history size
  • Active plugins
  • Custom data complexity

1.2 Set Up BigCommerce Store

  1. Create BigCommerce Account

    • Sign up at bigcommerce.com
    • Choose appropriate plan
    • Complete store setup wizard

  2. Configure Store Settings

    • Currency (must match WooCommerce)
    • Tax settings
    • Shipping zones
    • Payment gateways

  3. Get API Credentials

    • Go to Settings → API Accounts
    • Create API account
    • Note Store Hash and Access Token
    • Ensure required scopes are enabled:
      • store_v2_products
      • store_v2_customers
      • store_v2_orders
      • store_cart
      • store_checkout

1.3 Prepare WooCommerce

  1. Create REST API Keys

    • Go to WooCommerce → Settings → Advanced → REST API
    • Create new key with Read permissions
    • Note Consumer Key and Consumer Secret

  2. Verify REST API Access

    • Test connection: GET /wp-json/wc/v3/products?per_page=1
    • Should return product data

  3. Backup Your Store

    • Full WordPress backup
    • Database backup
    • File system backup

Phase 2: Migration Execution

2.1 Configure Migration Tool

Create .env file:

# WooCommerce
WC_URL=https://your-store.com
WC_CONSUMER_KEY=ck_xxxxx
WC_CONSUMER_SECRET=cs_xxxxx

# BigCommerce
BC_STORE_HASH=your-store-hash
BC_ACCESS_TOKEN=your-api-token

2.2 Migrate Categories

Categories must be migrated first (products depend on them):

npx tsx src/cli.ts migrate --type=categories

What happens:

  • Fetches all categories from WooCommerce
  • Transforms category data to BigCommerce format
  • Creates categories in BigCommerce
  • Preserves hierarchy (parent/child relationships)
  • Skips duplicates (idempotent)

Expected output:

Migrating categories...
[████████████████████] 100% 45/45 categories
Categories migrated successfully

2.3 Migrate Products

Migrate products (including variants):

npx tsx src/cli.ts migrate --type=products

What happens:

  • Fetches all products from WooCommerce
  • Transforms product data to BigCommerce format
  • Handles variants (variable products)
  • Uploads product images (references BC CDN URLs)
  • Creates products in BigCommerce
  • Matches by SKU or wc-{id} pattern (idempotent)

Expected output:

Migrating products...
[████████████████████] 100% 2847/2847 products
Products migrated successfully

Note: Products with >600 variants will show a warning (BC limit).

2.4 Migrate Customers

Migrate customer data:

npx tsx src/cli.ts migrate --type=customers

What happens:

  • Fetches all customers from WooCommerce
  • Transforms customer data to BigCommerce format
  • Creates customers in BigCommerce
  • Matches by email (idempotent)

Important: Customers will need to reset their passwords. BigCommerce doesn't support importing WooCommerce password hashes.

Expected output:

Migrating customers...
[████████████████████] 100% 12500/12500 customers
Customers migrated successfully
Note: Password reset required for all customers

2.5 Migrate Orders (Optional)

Migrate historical orders (optional):

npx tsx src/cli.ts migrate --type=orders

What happens:

  • Fetches orders from WooCommerce
  • Transforms order data to BigCommerce format
  • Creates orders in BigCommerce
  • Links to migrated products and customers

Note: Order migration is optional and can be time-consuming for large order histories.

Phase 3: Validation

3.1 Run Validation

Verify migration completeness:

npm run validate

What it checks:

  • Product counts (WC vs BC)
  • Category counts
  • Customer counts
  • Sample price accuracy
  • Image accessibility

Expected output:

Validating migration...
Products: 2847 (WC) = 2847 (BC)
Categories: 45 (WC) = 45 (BC)
Customers: 12500 (WC) = 12500 (BC)
Sample prices match
Images accessible

3.2 Manual Verification

Manually verify in BigCommerce admin:

  1. Check Products

    • Browse products in BC admin
    • Verify images load
    • Check variant options
    • Verify prices

  2. Check Categories

    • Verify category hierarchy
    • Check category descriptions
    • Verify product assignments

  3. Check Customers

    • Verify customer count
    • Check customer data (names, emails)
    • Note: Passwords need reset

Phase 4: Post-Migration

4.1 Configure BigCommerce

Complete store configuration:

  1. Payment Gateways

    • Configure payment methods
    • Test transactions

  2. Shipping

    • Set up shipping zones
    • Configure shipping methods
    • Set shipping rates

  3. Tax

    • Configure tax settings
    • Set up tax zones
    • Or integrate TaxJar/Avalara

  4. Store Settings

    • Store name and branding
    • Email templates
    • Checkout settings

4.2 Install BC Bridge Plugin

If using BC Bridge (headless commerce):

  1. Install BC Bridge plugin in WordPress
  2. Configure API credentials
  3. Test product display
  4. Test cart functionality
  5. Test checkout flow

See BC Bridge Installation Guide for details.

4.3 Customer Communication

Notify customers:

  1. Password Reset

    • Send password reset emails
    • Provide instructions
    • Set up password reset flow

  2. Account Access

    • Explain new login process
    • Provide support contact

  3. Order History

    • If orders migrated, explain access
    • Provide order lookup instructions

Phase 5: Go Live

5.1 Pre-Launch Checklist

  • All products migrated and verified
  • Categories organized correctly
  • Customers notified about password reset
  • Payment gateways configured and tested
  • Shipping methods configured
  • Tax settings configured
  • BC Bridge plugin installed (if using)
  • Storefront tested
  • Checkout tested end-to-end
  • SEO redirects configured (if needed)

5.2 Launch

  1. Switch DNS (if using custom domain)
  2. Activate BC Bridge (if using headless)
  3. Monitor for first 24-48 hours
  4. Support - Be ready for customer questions

Troubleshooting

See Troubleshooting Guide for common issues.

1. Pre-Engagement Qualification

1.1 Ideal Customer Profile (ICP)

Must Have

  • Current WooCommerce deployment with active transactions
  • $500K+ annual GMV
  • Business decision to migrate (not just exploring)
  • Budget aligned with implementation costs ($100K-$300K)
  • Timeline flexibility (16-20 weeks minimum)

Strong Indicators

  • 500+ blog posts on WordPress (content retention need)
  • B2B/wholesale revenue > 20% of GMV
  • Marketplace revenue > 30% of GMV
  • In-house dev team or agency relationship
  • Previous negative experience with Shopify or Shopify limitations awareness

Disqualifiers

  • GMV < $250K (better fit for standard Shopify)
  • No technical resources (can't maintain headless)
  • Timeline < 12 weeks (insufficient for quality migration)
  • Decision-maker prefers Shopify (don't fight the default)

Phase 4: Cutover & Launch

Duration: Weeks 15-16 | Objective: Execute go-live with minimal disruption

6.1 Pre-Launch Checklist

T-7 Days

  • Final data sync (delta migration)
  • All redirects uploaded and tested
  • DNS TTL reduced to 300 seconds
  • SSL certificate provisioned on Vercel
  • Customer activation emails scheduled
  • Rollback plan documented

T-1 Day

  • Final delta migration (orders, customers)
  • WooCommerce set to maintenance mode (or read-only)
  • All stakeholders notified of launch window
  • War room / command center established

6.2 Launch Sequence

HOUR 0: CUTOVER BEGINS ├── 0:00 - WooCommerce → Maintenance mode ├── 0:15 - Final data delta sync ├── 0:30 - Delta validation complete ├── 0:45 - DNS change initiated HOUR 1: PROPAGATION ├── 1:00 - Monitor DNS propagation (whatsmydns.net) ├── 1:30 - First traffic arriving at new site ├── 2:00 - Verify orders processing on BigCommerce HOUR 2-4: VALIDATION ├── 2:00 - Spot-check SEO (key pages indexing) ├── 3:00 - Verify integrations (ERP sync, emails) ├── 4:00 - Customer activation emails sent HOUR 4+: MONITORING └── Continuous monitoring for 48 hours

6.3 Rollback Procedure

Trigger Conditions:

  • Checkout failure rate > 10%
  • Payment processing failure
  • Critical integration down
  • Executive decision

Rollback Steps

  • Revert DNS to WooCommerce server
  • Disable maintenance mode on WooCommerce
  • Notify stakeholders of rollback
  • Export any orders placed on BigCommerce during window
  • Manually import those orders to WooCommerce
  • Schedule post-mortem within 48 hours
