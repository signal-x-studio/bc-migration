Migration Troubleshooting

Common issues and solutions during WooCommerce to BigCommerce migration.

Connection Issues

WooCommerce 401 Unauthorized

Symptoms:

  • Error: 401 Unauthorized when connecting to WooCommerce API
  • Assessment fails immediately

Solutions:

  1. Verify API credentials:

    • Check WC_CONSUMER_KEY and WC_CONSUMER_SECRET in .env
    • Ensure no extra spaces or quotes

  2. Check API permissions:

    • Go to WooCommerce → Settings → Advanced → REST API
    • Verify key has Read permissions
    • Regenerate key if needed

  3. Verify REST API is enabled:

    • Check WooCommerce settings
    • Ensure REST API is not disabled

  4. Check user permissions:

    • API key user must have admin or shop manager role
    • Verify user account is active

BigCommerce 401/403 Errors

Symptoms:

  • Error: 401 Unauthorized or 403 Forbidden when connecting to BigCommerce
  • validate-target command fails

Solutions:

  1. Verify credentials:

    • Check BC_STORE_HASH (should be alphanumeric, no dashes)
    • Check BC_ACCESS_TOKEN (should be long string)
    • Ensure no extra spaces

  2. Check API token scopes:

    • Go to BigCommerce → Settings → API Accounts
    • Verify token has required scopes:
      • store_v2_products
      • store_v2_customers
      • store_v2_orders
      • store_cart
      • store_checkout

  3. Regenerate token if needed:

    • Create new API account
    • Copy new Store Hash and Access Token
    • Update .env file

Connection Timeout

Symptoms:

  • Requests timeout after 30 seconds
  • Intermittent connection failures

Solutions:

  1. Check network connectivity
  2. Verify store URLs are correct
  3. Check firewall settings
  4. Try from different network

Migration Issues

Products Not Migrating

Symptoms:

  • Migration completes but products missing in BigCommerce
  • Product count mismatch

Solutions:

  1. Check migration logs:

    cat logs/migration-*.log | grep -i error

  2. Verify category migration:

    • Products require categories
    • Run category migration first

  3. Check for SKU conflicts:

    • BigCommerce requires unique SKUs
    • Products without SKUs get wc-{id} pattern
    • Check for duplicate SKUs

  4. Verify product data:

    • Check for required fields (name, price)
    • Verify product types are supported

Variants Not Migrating

Symptoms:

  • Variable products migrate but variants missing
  • Variant count mismatch

Solutions:

  1. Check variant limit:

    • BigCommerce has 600 variant limit per product
    • Products exceeding limit will show warning
    • Consider splitting into multiple products

  2. Verify variant attributes:

    • Variants require parent product attributes
    • Check attribute configuration in WooCommerce

  3. Check variant data:

    • Verify variant prices
    • Check for required variant fields

Categories Not Migrating

Symptoms:

  • Categories missing in BigCommerce
  • Category hierarchy broken

Solutions:

  1. Check for duplicate names:

    • BigCommerce requires unique category names
    • Duplicates are skipped (idempotent)

  2. Verify category data:

    • Check for required fields (name)
    • Verify parent category exists

  3. Check migration order:

    • Parent categories must exist before children
    • Migration handles this automatically

Customers Not Migrating

Symptoms:

  • Customer count mismatch
  • Customers missing in BigCommerce

Solutions:

  1. Check for duplicate emails:

    • BigCommerce requires unique emails
    • Duplicates are skipped (idempotent)

  2. Verify customer data:

    • Check for required fields (email)
    • Verify email format is valid

  3. Check customer count:

    • Verify WooCommerce customer count
    • Check for guest customers (may not migrate)

Rate Limit Errors

Symptoms:

  • Error: 429 Too Many Requests
  • Migration slows down or fails

Solutions:

  1. Rate limiting is automatic:

    • Tool handles rate limits automatically
    • Uses exponential backoff
    • Queues requests

  2. For very large migrations:

    • Migration will take longer
    • Check logs for progress
    • Consider migrating in batches

  3. Reduce batch size (if needed):

    • Edit migration code to reduce batch size
    • Default is 10 items per batch

Data Issues

Price Mismatches

Symptoms:

  • Prices don't match between WC and BC
  • Validation shows price differences

Solutions:

  1. Check currency settings:

    • BigCommerce currency must match WooCommerce
    • Verify currency in BC store settings

  2. Check for sale prices:

    • Sale prices migrate as separate field
    • Verify sale price display settings

  3. Check for currency conversion:

    • Ensure no currency conversion applied
    • Verify exchange rates

Image Issues

Symptoms:

  • Images not loading in BigCommerce
  • Image URLs broken

Solutions:

  1. Images use BigCommerce CDN:

    • Images are uploaded to BigCommerce
    • URLs point to BC CDN
    • Verify CDN access

  2. Check image formats:

    • Supported: JPG, PNG, GIF, WebP
    • Verify image files are accessible

  3. Check image sizes:

    • Very large images may fail
    • Check file size limits

Missing Data

Symptoms:

  • Some product data missing
  • Custom fields not migrated

Solutions:

  1. Check custom fields:

    • Custom fields may not migrate automatically
    • Use custom field mapping if needed

  2. Check meta data:

    • Some WooCommerce meta may not map to BC
    • Review data mapping documentation

  3. Check for required fields:

    • BigCommerce has required fields
    • Missing required fields cause migration to skip

Validation Issues

Count Mismatches

Symptoms:

  • Validation shows count differences
  • Products/categories/customers don't match

Solutions:

  1. Check for skipped items:

    • Review migration logs for skipped items
    • Check for error messages

  2. Verify source data:

    • Check WooCommerce counts
    • Verify API is returning all data

  3. Check for duplicates:

    • Duplicates are skipped (idempotent)
    • Verify duplicate detection logic

Sample Validation Failures

Symptoms:

  • Sample price validation fails
  • Sample data doesn't match

Solutions:

  1. Check sample size:

    • Default is 10 samples
    • Increase sample size for verification

  2. Check for data transformation:

    • Prices may be transformed
    • Verify transformation logic

  3. Check for rounding:

    • Price rounding may cause small differences
    • Verify rounding precision

Performance Issues

Slow Migration

Symptoms:

  • Migration takes very long
  • Progress seems stuck

Solutions:

  1. Check rate limiting:

    • Rate limits slow down migration
    • This is expected for large stores

  2. Check network:

    • Slow network affects migration speed
    • Verify network connectivity

  3. Check API response times:

    • Slow API responses slow migration
    • Check BigCommerce status page

Memory Issues

Symptoms:

  • Out of memory errors
  • Process crashes

Solutions:

  1. Increase Node.js memory:

    node --max-old-space-size=4096 dist/cli.js migrate

  2. Migrate in smaller batches:

    • Migrate by category
    • Migrate products in chunks

  3. Check for memory leaks:

    • Review migration code
    • Check for large data structures

Getting Help

If you encounter issues not covered here:

  1. Check migration logs: logs/migration-*.log
  2. Review Migration Architecture
  3. Check Data Mapping
  4. Contact BigCommerce support
