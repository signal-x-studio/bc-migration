Migration Wizard Guide

The Migration Wizard is an interactive, phase-based tool that guides you through migrating your WooCommerce store to BigCommerce. It breaks the migration into manageable phases and provides real-time progress tracking.

Overview

The Migration Wizard consists of four phases:

  1. Phase 1: Foundation - Set up category structure (Required)
  2. Phase 2: Core Data - Migrate products and customers (Required)
  3. Phase 3: Transactions - Migrate orders and coupons (Optional)
  4. Phase 4: Content - Migrate reviews, pages, and blog posts (Optional)

Getting Started

Prerequisites

  • WooCommerce store with REST API credentials
  • BigCommerce store with API credentials
  • Completed store assessment (recommended)

Accessing the Wizard

  1. Navigate to the Migrate page in the dashboard
  2. Click Start Migration Wizard
  3. Enter your WooCommerce and BigCommerce credentials
  4. Click Begin Migration

Wizard Interface

Progress Indicator

The top of the wizard shows:

  • Current phase number and name
  • Overall progress (completed phases / total phases)
  • Phase status indicators (pending, in progress, complete, locked)

Phase Navigation

You can navigate between phases using:

  • Previous Phase: Go back to review or modify previous phase
  • Next Phase: Proceed to next available phase
  • Phase Selector: Click on phase indicators to jump to specific phases

Phase Status

Each phase can have one of these statuses:

  • Pending: Not started yet
  • In Progress: Currently being worked on
  • Complete: Successfully completed
  • Locked: Cannot be started (dependencies not met)
  • Skipped: Optional phase that was skipped

Phase 1: Foundation

What It Does

Sets up your category structure in BigCommerce by migrating all WooCommerce categories.

How to Use

  1. Review the phase description and requirements
  2. Click Set Up Categories
  3. Monitor real-time progress
  4. Review the results summary:
    • Total categories created
    • Categories skipped (already existed)
    • Any errors encountered
    • Category ID mapping table
  5. Click Continue to Phase 2 when ready

Expected Duration

Typically 5-15 minutes depending on category count (usually 50-200 categories).

Phase 2: Core Data

What It Does

Migrates your product catalog and customer accounts to BigCommerce.

How to Use - Products

  1. Browse the category tree (from Phase 1)
  2. Select a category to see its product count
  3. Click Migrate Products for the selected category
  4. Monitor real-time progress:
    • Current product being migrated
    • Progress bar and percentage
    • Products completed / total
  5. Review results for each category
  6. Repeat for additional categories

How to Use - Customers

  1. Scroll to the Customers section
  2. Click Migrate Customers
  3. Monitor real-time progress
  4. Review customer migration statistics

Expected Duration

Varies significantly based on data volume:

  • Products: 1-5 minutes per 100 products
  • Customers: 1-3 minutes per 100 customers

Phase 3: Transactions

What It Does

Migrates order history and discount codes to BigCommerce.

Dependencies

Requires Phase 2 (Core Data) to be complete. The wizard will show a warning if dependencies aren't met.

How to Use - Orders

  1. Verify Phase 2 is complete
  2. Click Migrate Orders in the Orders section
  3. Monitor real-time progress
  4. Review order migration statistics

How to Use - Coupons

  1. Click Migrate Coupons in the Coupons section
  2. Monitor real-time progress
  3. Review coupon migration statistics

Skipping This Phase

If you don't need to migrate orders or coupons, you can skip this phase by clicking Skip Phase.

Expected Duration

Varies based on order count:

  • Orders: 2-5 minutes per 100 orders
  • Coupons: 1-2 minutes per 50 coupons

Phase 4: Content

What It Does

Migrates product reviews, CMS pages, and blog posts to BigCommerce.

Dependencies

Reviews require Phase 2 (Core Data) to be complete. Pages and blog posts can be migrated independently.

How to Use - Reviews

  1. Verify Phase 2 is complete (for product associations)
  2. Click Migrate Reviews in the Reviews section
  3. Monitor real-time progress
  4. Review migration statistics

How to Use - Pages

  1. Click Migrate Pages in the Pages section
  2. Monitor real-time progress
  3. Review migration statistics

How to Use - Blog Posts

  1. Click Migrate Blog Posts in the Blog section
  2. Monitor real-time progress
  3. Review migration statistics

Skipping This Phase

If you don't need to migrate content, you can skip this phase by clicking Skip Phase.

Expected Duration

Varies based on content volume:

  • Reviews: 1-2 minutes per 100 reviews
  • Pages: 30 seconds per 10 pages
  • Blog Posts: 1-3 minutes per 50 posts (includes image processing)

Wizard Features

Real-time Progress

Each phase shows real-time progress updates:

  • Current item being processed
  • Progress bar with percentage
  • Items completed / total items
  • Estimated time remaining (when available)

Error Handling

The wizard handles errors gracefully:

  • Individual item failures don't stop the migration
  • Errors are logged and displayed in results
  • You can retry failed items or continue with successful ones

State Persistence

Your progress is automatically saved:

  • Phase completion status
  • ID mappings (categories, products, customers)
  • Migration statistics
  • You can resume from where you left off

ID Mappings

The wizard maintains ID mappings between platforms:

  • Category Mapping: WC Category ID → BC Category ID
  • Product Mapping: WC Product ID → BC Product ID
  • Customer Mapping: WC Customer ID → BC Customer ID

These mappings are used in subsequent phases to maintain relationships.

Best Practices

Before Starting

  • Run Assessment: Complete a store assessment to identify potential issues
  • Backup Stores: Backup both WooCommerce and BigCommerce stores
  • Test Credentials: Verify API credentials work correctly
  • Review Data: Clean up unnecessary data before migration

During Migration

  • Monitor Progress: Watch for errors and warnings
  • Don't Interrupt: Let each phase complete before moving on
  • Review Results: Check statistics and mappings after each phase
  • Take Notes: Document any issues or manual steps needed

After Migration

  • Verify Data: Spot-check migrated data in BigCommerce
  • Test Functionality: Test key store functions (browse, cart, checkout)
  • Update Links: Review and update internal links if needed
  • Communicate: Notify customers about password reset requirement

Troubleshooting

Wizard Won't Start

  • Verify API credentials are correct
  • Check that both stores are accessible
  • Review browser console for errors
  • Try refreshing the page

Phase Stuck or Slow

  • Check network connection
  • Review API rate limits
  • Check server logs for errors
  • Consider migrating in smaller batches

Errors During Migration

  • Review error messages in the results
  • Check migration logs for details
  • Verify data quality in source store
  • Retry failed items if possible

Can't Proceed to Next Phase

  • Verify current phase is marked as complete
  • Check that all required steps are done
  • Review dependency requirements
  • Try refreshing the wizard state

Keyboard Shortcuts

  • Esc: Close dialogs or cancel operations
  • Enter: Submit forms or confirm actions
  • Arrow Keys: Navigate between phases (when available)
