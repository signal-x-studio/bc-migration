Phase 4: Content - Reviews, Pages, and Blog

The Content phase migrates your content assets: product reviews, CMS pages, and blog posts. This is an optional phase that depends on Phase 2 (Core Data) being completed first for reviews.

Overview

This phase migrates:

  • Product Reviews: Customer reviews with ratings, comments, and product associations
  • CMS Pages: Static pages (About, Contact, Terms, etc.)
  • Blog Posts: Blog articles with content, images, and metadata

Dependencies

Requires Phase 2 (Core Data) for Reviews:

  • Product reviews need products to be migrated (for product associations)
  • Product ID mapping from Phase 2 is used to link reviews to products
  • Pages and blog posts can be migrated independently

Reviews Migration

What Gets Migrated

  • Review Content: Review title, comment text, rating
  • Product Association: Links review to migrated product (using product ID mapping)
  • Reviewer Information: Reviewer name, email (if available)
  • Review Status: Approved, pending, or spam status
  • Review Date: Original review date

Review Migration Process

  1. Fetch Reviews: Retrieves all product reviews from WooCommerce
  2. Resolve Product: Maps review to product using product ID mapping
  3. Transform Review: Converts WooCommerce review format to BigCommerce format
  4. Create Review: Creates review in BigCommerce via API

Product Resolution

Reviews are linked to products using the product ID mapping from Phase 2:

// For each review
const productId = productIdMapping[wcReview.product_id];
if (productId) {
  // Link review to migrated product
  reviewPayload.product_id = productId;
} else {
  // Skip review if product not found
  stats.skipped++;
  continue;
}

Review Transformation

Reviews are transformed to match BigCommerce review structure:

const reviewPayload = {
  product_id: productId, // From product ID mapping
  title: wcReview.title || wcReview.review.substring(0, 50),
  text: wcReview.review,
  status: mapReviewStatus(wcReview.status), // 'approved', 'pending', 'disapproved'
  rating: wcReview.rating,
  reviewer: {
    name: wcReview.reviewer_name,
    email: wcReview.reviewer_email
  },
  date_reviewed: wcReview.date_created
};

Review Migration Statistics

After completion, you'll receive:

  • Total: Total reviews found in WooCommerce
  • Migrated: Reviews successfully created in BigCommerce
  • Skipped: Reviews for products not found or invalid
  • Failed: Reviews that encountered errors

Pages Migration

What Gets Migrated

  • Page Content: Page title, content (HTML), excerpt
  • Page Metadata: Slug, author, publish date, modified date
  • Page Status: Published, draft, or private status
  • Page Template: Template assignment (if applicable)
  • Featured Image: Page featured image

Page Migration Process

  1. Fetch Pages: Retrieves all pages from WooCommerce
  2. Transform Page: Converts WooCommerce page format to BigCommerce format
  3. Handle Content: Processes HTML content and images
  4. Create Page: Creates page in BigCommerce via API

Page Transformation

Pages are transformed to match BigCommerce page structure:

const pagePayload = {
  name: wcPage.title.rendered,
  content: wcPage.content.rendered,
  meta_description: wcPage.excerpt.rendered,
  is_visible: wcPage.status === 'publish',
  sort_order: wcPage.menu_order,
  url: wcPage.slug
};

Page Migration Statistics

After completion, you'll receive:

  • Total: Total pages found in WooCommerce
  • Migrated: Pages successfully created in BigCommerce
  • Skipped: Pages that already existed (by slug)
  • Failed: Pages that encountered errors

Blog Posts Migration

What Gets Migrated

  • Post Content: Post title, content (HTML), excerpt
  • Post Metadata: Slug, author, publish date, modified date
  • Post Status: Published, draft, or private status
  • Categories: Post categories (if applicable)
  • Tags: Post tags
  • Featured Image: Post featured image
  • Post Images: Images embedded in post content

Blog Migration Process

  1. Fetch Posts: Retrieves all blog posts from WooCommerce
  2. Process Content: Extracts and processes HTML content
  3. Handle Images: Downloads and uploads images to BigCommerce
  4. Transform Post: Converts WooCommerce post format to BigCommerce format
  5. Create Post: Creates blog post in BigCommerce via API

Content Processing

Blog post content is processed to handle:

  • Image URLs: Replaces WooCommerce image URLs with BigCommerce URLs
  • Internal Links: Updates internal links to match new URL structure
  • HTML Formatting: Preserves HTML structure and formatting
  • Embedded Media: Handles embedded videos, galleries, etc.

Blog Migration Statistics

After completion, you'll receive:

  • Total: Total blog posts found in WooCommerce
  • Migrated: Posts successfully created in BigCommerce
  • Skipped: Posts that already existed (by slug)
  • Failed: Posts that encountered errors

CLI Usage

# Migrate all content
npm run migrate -- --phase content \
  --wc-url https://your-store.com \
  --wc-key YOUR_KEY \
  --wc-secret YOUR_SECRET \
  --bc-hash YOUR_HASH \
  --bc-token YOUR_TOKEN

# Migrate specific content types
npm run migrate -- --phase content --reviews-only
npm run migrate -- --phase content --pages-only
npm run migrate -- --phase content --blog-only

Important Considerations

Content Formatting

  • HTML Content: HTML is preserved but may need manual review
  • Image URLs: Images are downloaded and re-uploaded to BigCommerce
  • Internal Links: Links may need to be updated to match new URL structure

Review Associations

  • Reviews are only migrated if the associated product exists
  • Reviews for products not found are skipped
  • Consider migrating products before reviews for complete coverage

URL Structure

  • Page and blog post URLs (slugs) are preserved when possible
  • URL conflicts are resolved by appending numbers
  • Review and update URLs after migration

Next Steps

After completing all phases, you can:

  • Review migrated content in BigCommerce
  • Update URLs and internal links as needed
  • Test content display and functionality
  • Proceed with go-live preparation

Troubleshooting

Reviews Not Appearing

  • Verify products were migrated in Phase 2
  • Check that product ID mapping is correct
  • Review review status (only approved reviews may be visible)

Content Formatting Issues

  • Review HTML content in BigCommerce editor
  • Check that images uploaded correctly
  • Update internal links manually if needed

Image Issues

  • Verify image URLs are accessible
  • Check image file sizes and formats
  • Review image upload errors in logs

URL Conflicts

  • Review slug conflicts in migration logs
  • Update URLs manually if needed
  • Set up redirects for old URLs

Best Practices

  • Complete Core Data First: Ensure products are migrated before reviews
  • Review Content: Check migrated content for formatting issues
  • Update Links: Review and update internal links after migration
  • Test Display: Verify content displays correctly in BigCommerce
  • Handle Images: Review image uploads and update URLs if needed
  • Backup: Always backup your BigCommerce store before migration
