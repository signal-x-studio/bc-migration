Overview

The WordPress Content Component system enables seamless integration of WordPress blog posts, pages, and content blocks directly into BigCommerce headless storefronts built with Catalyst, Stencil, or Makeswift.

This capability is central to the "Keep WordPress" migration strategy, allowing merchants to retain their WordPress content investment while migrating commerce functionality to BigCommerce.

Strategic Value

Rather than migrating content to BigCommerce's native CMS, merchants can continue using WordPress for content management while displaying that content seamlessly within their BigCommerce storefront. This preserves SEO equity, editorial workflows, and plugin investments.

Component Inventory

WPContentRenderer

Core rendering component that transforms WordPress HTML content and applies theme-specific Tailwind CSS styling. Handles all HTML elements including headings, paragraphs, lists, blockquotes, code blocks, images, and Gutenberg blocks.

File: components/wordpress/WPContentRenderer.tsx

Props: content (HTML string), theme (catalyst|stencil|makeswift), className

WPContentBlock

Full-featured content block that fetches WordPress content by ID and renders with featured image, metadata (date, author, categories), and styled content body. Supports both posts and pages.

File: components/wordpress/WPContentBlock.tsx

Props: contentId, contentType (post|page), theme, showMeta, showFeaturedImage

WPPostList

Paginated content browser with grid/list view modes. Displays post thumbnails, titles, dates, and excerpts. Supports selection callbacks for content picker interfaces.

File: components/wordpress/WPPostList.tsx

Props: contentType, perPage, onSelect, selectedId

WPPostSelector

Dropdown selector component for choosing WordPress posts or pages. Includes search functionality and thumbnail previews. Ideal for Makeswift visual builder integration.

File: components/wordpress/WPPostSelector.tsx

Props: value, onChange, contentType, onContentTypeChange

Usage Examples

Basic Content Rendering

import { WPContentRenderer } from '@/components/wordpress/WPContentRenderer';

// Render WordPress HTML with Catalyst styling
<WPContentRenderer
  content={post.content.rendered}
  theme="catalyst"
/>

Full Content Block with Metadata

import { WPContentBlock } from '@/components/wordpress/WPContentBlock';

// Fetch and display a WordPress post by ID
<WPContentBlock
  contentId={123}
  contentType="post"
  theme="makeswift"
  showMeta={true}
  showFeaturedImage={true}
/>

Content Picker for Makeswift

import { WPPostSelector } from '@/components/wordpress/WPPostSelector';

// Dropdown selector for content selection
<WPPostSelector
  value={selectedPost}
  onChange={setSelectedPost}
  contentType="post"
  onContentTypeChange={setContentType}
/>

Browsable Content Grid

import { WPPostList } from '@/components/wordpress/WPPostList';

// Paginated grid of WordPress posts
<WPPostList
  contentType="post"
  perPage={9}
  onSelect={(item) => router.push(`/blog/${item.slug}`)}
/>

WordPress REST API Integration

Components communicate with WordPress via internal API routes that proxy to the WordPress REST API:

Internal Route WordPress Endpoint Purpose
/api/wp/posts /wp-json/wp/v2/posts Fetch posts with embeds
/api/wp/pages /wp-json/wp/v2/pages Fetch pages with embeds
/api/wp/content /wp-json/wp/v2/{type}/{id} Fetch single content item

WordPress Connection Context

import { useWPConnection } from '@/lib/contexts/WPConnectionContext';

const { siteUrl, isConnected } = useWPConnection();

// siteUrl: The WordPress site URL (e.g., "https://myblog.com")
// isConnected: Boolean indicating valid connection

Integration with Migration Strategy

The WordPress Content Components support the broader "Keep WordPress" migration strategy:

SEO Preservation

WordPress content URLs remain authoritative, preventing link equity loss

Content Continuity

Existing WordPress content appears seamlessly in BigCommerce storefront

Editorial Independence

Content team continues using WordPress admin without retraining

Plugin Compatibility

WordPress content plugins (Yoast, ACF) continue functioning

Makeswift Integration

The WPPostSelector component is specifically designed for drag-and-drop integration with Makeswift's visual builder. Content editors can select WordPress posts/pages directly within the Makeswift interface, with preview thumbnails and search functionality.
