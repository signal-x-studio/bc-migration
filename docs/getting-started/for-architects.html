Getting Started for Solution Architects

This guide covers system design, integration patterns, and architectural considerations for WooCommerce to BigCommerce migrations.

System Architecture

The migration system implements a headless commerce architecture:

  • Frontend: WordPress (or custom frontend)
  • Backend: BigCommerce (product data, inventory, checkout)
  • Integration: BC Bridge plugin or custom API integration

Migration Paths

BC Bridge

WordPress plugin that connects WordPress to BigCommerce:

  • Real-time API calls
  • Intelligent caching
  • Theme-agnostic

Headless

Custom frontend with BigCommerce backend:

  • Full control over frontend
  • Modern tech stack
  • Requires frontend development

Stencil

BigCommerce Stencil theme:

  • Native BigCommerce theme
  • Full feature support
  • Requires theme development

Integration Patterns

API Integration

Direct REST API integration with:

  • Rate limiting
  • Retry logic
  • Error handling

Caching Strategy

Multi-layer caching:

  • API response caching
  • Object cache (WordPress)
  • CDN caching
