Getting Started for Merchants

Welcome! This guide helps you migrate your WooCommerce store to BigCommerce without rebuilding your website.

What is BC Migration?

BC Migration is a tool that helps you upgrade from WooCommerce to BigCommerce as your commerce backend while keeping your WordPress site intact. You keep your:

  • WordPress content and pages
  • Theme and design
  • SEO rankings
  • Existing plugins (for non-commerce features)

Only the commerce engine changesâ€”everything else stays the same.

Why Migrate?

WooCommerce limitations:

  • Doesn't scale well beyond ~$5M in annual sales
  • Requires significant infrastructure investment to grow
  • Plugin conflicts and maintenance burden

BigCommerce benefits:

  • Built-in scalability and reliability
  • No infrastructure management needed
  • Enterprise-grade features out of the box
  • Better performance and uptime

Quick Start

Step 1: Assess Your Store

Run an assessment to see if your store is ready for migration:

npm run assess

This analyzes your store and generates a readiness report showing:

  • What data will be migrated
  • Any potential issues or blockers
  • Estimated migration time
  • Complexity score

Step 2: Review Your Report

Check the assessment report in ./reports/ to understand:

  • GREEN - Your store is ideal for automated migration
  • YELLOW - Migration is feasible with some manual configuration
  • RED - High complexity, may need technical review

Step 3: Plan Your Migration

Review the migration guide to understand:

  • What data gets migrated (products, customers, orders)
  • What stays in WordPress (content, pages, blog)
  • Timeline and process

Step 4: Set Up BigCommerce

  1. Create a BigCommerce account
  2. Get your API credentials (Store Hash and Access Token)
  3. Configure your store settings

Step 5: Run the Migration

npm run migrate

The tool will:

  1. Migrate categories
  2. Migrate products (with variants)
  3. Migrate customers
  4. Migrate orders (optional)

Step 6: Validate Results

npm run validate

This verifies that all data migrated correctly.

What Gets Migrated?

Data Type Migrated? Notes
Products Yes Including variants and images
Categories Yes With hierarchy preserved
Customers Yes Password reset required
Orders Optional Historical orders
Content/Pages No Stays in WordPress
Blog Posts No Stays in WordPress
Media No Images served from BigCommerce CDN

Common Questions

Will my site go down during migration?

No. The migration runs separately from your live site. Your WooCommerce store continues operating normally until you're ready to switch.

How long does migration take?

Depends on your store size:

  • Small stores (< 1,000 products): 1-2 hours
  • Medium stores (1,000-10,000 products): 2-4 hours
  • Large stores (> 10,000 products): 4-8 hours

Will I lose my SEO rankings?

No. Your WordPress URLs and content stay the same. The BC Bridge plugin ensures your product URLs continue working.

What about my existing plugins?

Non-commerce plugins (like contact forms, SEO tools) continue working. Commerce-related plugins are replaced by BigCommerce features.

Do I need technical knowledge?

Basic technical knowledge helps, but the tool is designed to be straightforward. You'll need to:

  • Run command-line tools
  • Get API credentials from WooCommerce and BigCommerce
  • Follow step-by-step guides
