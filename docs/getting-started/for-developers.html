Getting Started for Developers

Technical setup and integration guide for the BC Migration tooling.

Prerequisites

  • Node.js 18+
  • npm 9+
  • WooCommerce REST API credentials (Consumer Key/Secret)
  • BigCommerce API credentials (Store Hash/Access Token)

Installation

1. Clone and Install

git clone <repo-url>
cd bc-migration
npm install

2. Configure Environment

cp .env.example .env

Edit .env with your credentials:

# WooCommerce
WC_URL=https://your-store.com
WC_CONSUMER_KEY=ck_xxxxx
WC_CONSUMER_SECRET=cs_xxxxx

# BigCommerce
BC_STORE_HASH=your-store-hash
BC_ACCESS_TOKEN=your-api-token

3. Test Connections

# Test BigCommerce connection
npm run validate-target

# Test WooCommerce connection
npm run assess

Quick Start

Run Assessment

npm run assess

# Or with explicit credentials
npx tsx src/cli.ts assess --url https://store.com --key ck_xxx --secret cs_xxx

Run Migration

# Migrate all data types
npm run migrate

# Migrate specific type
npx tsx src/cli.ts migrate --type=products
npx tsx src/cli.ts migrate --type=categories
npx tsx src/cli.ts migrate --type=customers
npx tsx src/cli.ts migrate --type=orders

# Delta migration (only changed items)
npx tsx src/cli.ts migrate --delta

Validate Results

npm run validate

# With custom sample size
npx tsx src/cli.ts validate --sample 20

Architecture Overview

Assessment Engine

Analyzes WooCommerce stores for migration readiness:

  • Scale metrics (product count, variant count, etc.)
  • Complexity scoring
  • Plugin compatibility analysis
  • SEO structure analysis

Migration System

Transforms and migrates data from WooCommerce to BigCommerce:

  • Transformers - Convert WC data format to BC format
  • Migrators - Handle API calls and rate limiting
  • State Tracker - Track migration progress for delta syncs

Validation System

Verifies migration completeness:

  • Count comparisons (WC vs BC)
  • Sample data validation
  • Image accessibility checks

API Integration

WooCommerce REST API

Used for reading source data:

  • GET /wp-json/wc/v3/products - Products
  • GET /wp-json/wc/v3/products/categories - Categories
  • GET /wp-json/wc/v3/customers - Customers
  • GET /wp-json/wc/v3/orders - Orders

BigCommerce REST API

Used for writing target data:

  • POST /v3/catalog/products - Create products
  • POST /v3/catalog/categories - Create categories
  • POST /v3/customers - Create customers
  • POST /v3/orders - Create orders

Rate Limits:

  • 150 requests per 30 seconds
  • 10 items per batch operation
  • 600 variants per product (hard limit)

Key Concepts

Idempotent Operations

All migration operations are idempotent—safe to re-run without creating duplicates:

  • Categories: Skipped if name already exists
  • Products: Matched by SKU or wc-{id} pattern
  • Customers: Matched by email

Rate Limiting

BigCommerce API rate limiting is handled automatically:

  • Uses Bottleneck library
  • Exponential backoff on rate limit errors
  • Request queuing for bursts

Error Handling

  • Structured logging with Pino
  • Retry logic with exponential backoff
  • Graceful degradation on non-critical errors

Troubleshooting

Connection Errors

WooCommerce 401 Unauthorized

  • Verify WC_CONSUMER_KEY and WC_CONSUMER_SECRET
  • Ensure REST API is enabled
  • Check API key permissions (read required)

BigCommerce 401/403

  • Verify BC_STORE_HASH and BC_ACCESS_TOKEN
  • Ensure token has required scopes

Rate Limiting

Rate limit errors are handled automatically, but you can:

  • Check logs/migration-*.log for timing details
  • Reduce batch sizes for very large migrations

Missing Data

  • Run npm run validate to check for discrepancies
  • Check logs for transformation warnings
  • Products without SKUs get auto-generated SKUs (wc-{id})
