Getting Started for WordPress Developers

BC Bridge is a WordPress plugin that connects your WordPress site to BigCommerce, enabling you to display and sell products without rebuilding your theme. This guide will help you get started with BC Bridge development.

What is BC Bridge?

BC Bridge is a headless commerce solution that:

  • Connects WordPress to BigCommerce via REST API
  • Displays products, categories, and cart functionality in WordPress
  • Uses intelligent caching to minimize API calls
  • Works with any WordPress theme
  • Supports embedded checkout or redirect checkout

Key Concepts

Headless Commerce

BC Bridge implements a headless commerce architecture:

  • Frontend: WordPress handles the presentation layer
  • Backend: BigCommerce handles product data, inventory, and checkout
  • API: Real-time communication via BigCommerce REST API

No Product Sync

Unlike traditional e-commerce plugins, BC Bridge does not sync products to WordPress:

  • Product data stays in BigCommerce
  • WordPress fetches data on-demand via API
  • Intelligent caching reduces API calls
  • No database bloat from product data

Template System

BC Bridge uses WordPress template hierarchy:

  • Default templates in bc-bridge/templates/
  • Theme overrides in your-theme/bc-bridge/
  • Standard WordPress template functions
  • Customizable via hooks and filters

Requirements

WordPress Requirements

  • WordPress 6.2 or higher
  • PHP 8.1 or higher
  • SSL certificate (required for checkout)

BigCommerce Requirements

  • BigCommerce store with API access
  • API credentials (Store Hash and Access Token)
  • Required API scopes:
    • Products: Read-only
    • Carts: Modify
    • Checkouts: Modify
    • Store Information: Read-only

Installation

From ZIP (Production)

  1. Download bc-bridge-x.x.x.zip from releases
  2. Go to Plugins > Add New > Upload Plugin
  3. Upload the ZIP file
  4. Activate the plugin

From Source (Development)

cd wp-content/plugins
git clone https://github.com/bigcommerce/bc-bridge.git
cd bc-bridge
make install

Build Commands

# Install dependencies
make install

# Build for production
make build

# Create distribution ZIP
make zip

# Deploy to test site
WP_PLUGINS_DIR=/path/to/plugins make deploy

# Create symlink for development
WP_PLUGINS_DIR=/path/to/plugins make deploy-link

# Watch for changes
make watch

# Run linters
make lint

# Bump version
make version NEW_VERSION=1.1.0

Configuration

Get BigCommerce API Credentials

  1. Log into your BigCommerce admin
  2. Go to Settings > API Accounts > Create API Account
  3. Name it "BC Bridge"
  4. Select required scopes:
    • Products: Read-only
    • Carts: Modify
    • Checkouts: Modify
    • Store Information: Read-only
  5. Copy the Store Hash and Access Token

Configure Plugin

  1. Go to BC Bridge > Settings in WordPress admin
  2. Enter your Store Hash
  3. Enter your Access Token
  4. Click "Test Connection" to verify
  5. Save settings

Basic Usage

Automatic Routes

BC Bridge automatically creates these routes:

  • Shop: yoursite.com/shop/
  • Product: yoursite.com/product/product-name/
  • Category: yoursite.com/product-category/category-name/
  • Cart: yoursite.com/cart/
  • Checkout: yoursite.com/checkout/

Shortcodes

Use shortcodes to display products anywhere:

// Product grid (12 products, 4 columns)
[bc_products limit="12" columns="4"]

// Products from specific category
[bc_products category="clothing" limit="8"]

// Single product by ID
[bc_product id="123"]

// Single product by slug
[bc_product slug="awesome-product"]

// Shopping cart
[bc_cart]

// Checkout
[bc_checkout]

PHP Functions

// Get API client instance
$api = bc_bridge()->api;

// Get products
$products = $api->get_products([
    'limit' => 12,
    'page' => 1,
    'categories:in' => 123,
]);

// Get single product
$product = $api->get_product(123);

// Get cart
$cart = bc_bridge()->cart->get_cart();

// Add to cart
$result = bc_bridge()->cart->add_item($product_id, $quantity, $variant_id);

// Get checkout URL
$checkout_url = bc_bridge()->cart->get_checkout_url();

// Clear cache
BC_Bridge_Cache::clear_all();
BC_Bridge_Cache::clear_group('products');

Core Classes

BC_Bridge

Main plugin class. Singleton pattern.

$bridge = BC_Bridge::instance();
$api = $bridge->api;
$cart = $bridge->cart;

BC_Bridge_API_Client

Handles all BigCommerce API communication.

$api = bc_bridge()->api;
$products = $api->get_products(['limit' => 12]);
$product = $api->get_product(123);

BC_Bridge_Cart

Manages shopping cart operations.

$cart = bc_bridge()->cart;
$cart_data = $cart->get_cart();
$result = $cart->add_item($product_id, $quantity, $variant_id);

BC_Bridge_Cache

Handles caching of API responses.

BC_Bridge_Cache::clear_all();
BC_Bridge_Cache::clear_group('products');
